在这里我们要先知道，程序是怎么知道要以64位模式运行还是以32位模式运行的；寄存器中有一个cs寄存器，cs = 0x23代表32位模式，cs = 0x33代表64位模式，而cs寄存器就是通过上面提到的retfq汇编指令来修改

retfq有两步操作，ret以及set cs，所以执行retfq会跳转到rsp同时将cs设置为[rsp+0x8]，我们只需要事先在ret位置写入32位的shellcode就可以执行了，但是这里有一点需要注意的是，retfq跳转过去的时候程序已经切换成了32位模式，所以地址解析也是以32位的规则来的，所以原先的rsp = 0x7ffe530d01b8会被解析成esp = 0x530d01b8

因为call的是rbx，所以rbx中存储着shellcode执行的起始地址，将rbx pop 给rax，就可以用rax+偏移来得到push rdx;pop rdx;指令的地址，进而进行异或处理（其实不是很懂为什么不直接用rbx做基址）

NX保护

作用

将数据（堆、栈）所在内存页标识为不可执行，当程序尝试在数据页面上执行指令时，CPU会抛出异常而不是去执行指令。

编译选项

关闭：-z execstack

开启：-z noexecstack

PIE保护

作用

使程序每次加载的基址随机化

编译选项

关闭：-no-pie

开启：-pie-fPIC

Canary保护

作用

函数开始执行时往栈上插入Canary值，并在函数返回的时候检查Canary的值是否有变动，如果有则调用 __stack_check_fail 函数强行停止程序。

编译选项

关闭：-fno-stack-protector

部分启用（只为局部变量中含有 char 的函数插入Canary）：-fstack-protector

全局启用（为所有函数插入Canary）：-fstack-protector-all

Fortify保护

作用

主要用来防止格式化字符串漏洞。包含 %n 的格式化字符串不能位于程序内存中的可写地址。当使用位置参数时，必须使用范围内的所有参数（例如要使用 %7$x ，则必须同时使用 1$，2$，3$……）

编译选项

关闭：-D_FORTIFY_SOURCE = 0

开启：-D_FORTIFY_SOURCE = 2

RELRO保护

作用

设置符号重定向表为只读并在程序启动时就解析并绑定所有动态符号，从而减少对GOT表的攻击

编译选项

开启（部分）：-z lazy

开启（完全）：-z now

发表于 2021-12-16 更新于 2021-12-18

知识点

栈中的数组越界

栈的增长是由高地址往低地址增长的，数组的存储是由低地址往高地址的。

如果存在能使用数组元素但未对其下标进行检查的情况就可能发生数组越界，导致栈中的其他内容被泄漏或者被修改，同时如果访问负数的数组下标可以访问到更低地址的栈空间。

SROP基本原理和利用

SROP_「二进制安全pwn基础」 - 网安 (wangan.com)

[BUUCTF]PWN——ciscn_2019_es_7[详解]

SROP利用了Linux系统信号处理过程中的漏洞：在信号处理过程中会将用户态上下文环境及寄存器的值保存在用户态的栈中，处理完后再读取栈中的数据恢复寄存器的值。其中“处理完后再读取栈中的数据恢复寄存器中的值”就是 sigreturn 系统调用的作用，它会读取当前栈空间中的数据并传给相应寄存器。

sigreturn 系统调用会改变所有寄存器的值，包括RSP/ESP与RBP/EBP的值，所以虽然可以利用SROP来实现任何想要的系统调用，但是同时其会改变栈顶指针，有时候这不是我们想要的。

综上，SROP的攻击思路就是利用栈溢出设置返回地址为 sigreturn 系统调用的gadget，然后在栈上布置对应寄存器的数据，待 sigreturn 系统调用结束，所以寄存器的值都会变成我们布置在栈上的值。

完成一次 SROP 攻击需要具备四个条件：

  1. 存在栈溢出漏洞，使得栈上的数据可控
  2. 需要知道 ‘/bin/sh’ 的地址，如果是自己构造在栈上的，那就需要知道栈的地址
  3. 需要知道 syscall 指令在内存中的地址
  4. 需要知道 sigreturn 系统调用的内存地址（这个单独的gadget并不是必须的，因为 sigreturn 也是一个系统调用，只要令 rax = 15 或 eax = 77 ，然后再调用 syscall 指令，就相当于调用了 sigreturn ）
